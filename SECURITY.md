# Sécurité

Si vous découvrez une vulnérabilité de sécurité, merci d’éviter d’ouvrir une issue publique.

## Signalement

- Contactez le mainteneur du dépôt (GitHub) avec un maximum de détails (étapes, impact, version/commit).
- Si la fonctionnalité “Security advisories” est activée, privilégiez un rapport privé via GitHub.

## Bonnes pratiques

- Ne commitez jamais de secrets : fichiers `.env`, clés, mots de passe, tokens.
